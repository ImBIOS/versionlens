# Task: Multi-registry Support

## Description
Add support for Cargo.toml (crates.io), go.mod (Go), pyproject.toml (PyPI), Gemfile (RubyGems), pubspec.yaml (pub.dev).

## Acceptance Criteria
- [ ] Parse Cargo.toml to extract crate names
- [ ] Parse go.mod to extract module names
- [ ] Parse pyproject.toml to extract package names
- [ ] Implement registry clients for all 5 additional registries
- [ ] Auto-detect file type and use appropriate parser/client

## Technical Details
- Registry APIs:
  - crates.io: GET /api/v1/crates/{name} → {"crate": {"max_version": "x.y.z"}}
  - PyPI: GET /pypi/{package}/json → {"info": {"version": "x.y.z"}}
  - Go: proxy.golang.org/{module}/@v/list
  - RubyGems: GET /gems/{gem}.json → {"version": "x.y.z"}
  - pub.dev: GET /api/packages/{package} → {"latest": {"version": "x.y.z"}}
- Create registry trait for consistent interface
